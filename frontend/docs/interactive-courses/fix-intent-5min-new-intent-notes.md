# fix-intent-5min · 신규 인텐트 판정 메모

> Stage 1에서 유저가 식별한 오분류(`재수강` 라벨 / "시험 못보면 어떻게
> 되는거야")를 해소하기 위해 **어떤 새 인텐트를 만들면 되는가**에 대한
> 기준 메모. 추후 `workContent` / `result` 등 자유 입력 필드의 LLM 판정
> 프롬프트에서 컨텍스트로 활용한다.

## 왜 New Intent가 필요한가

유저 발화의 실제 주제는 "시험에 결시했을 때 어떻게 처리되는가"인데,
현 시스템 인텐트 중 이 주제를 담당하는 게 없음. 그래서 엉뚱한 `재수강`
으로 분류됐고, 이를 바로잡으려면 **결시 주제 전용 인텐트를 새로 만들어야**
한다.

## 모범 인텐트 이름: **시험 결시**

- "결시"는 학사 맥락에서 쓰이는 공식 용어 → 분류기 안정적
- "시험"을 앞에 두어 휴강·행사 등 비-시험 결시와 혼선 방지
- 답변 주제(병결 증빙 · 담당 교수 연락 · 대체 시험 · 학사 규정)가 모두
  "시험 결시 후 처리" 한 주제에 수렴 → 단일 인텐트로 소화
- 기존 네이밍 톤(`재수강`·`학사 일정`·`장학금`·`편입`)과 일치

## 판정 정책 (넓게)

"시험 결시"가 모범이지만 **LLM 판정은 넓게** 간다. 유저가 아래처럼
비슷한 주제를 담은 네이밍을 가져오면 통과시킨다.

### 통과 가능한 네이밍 예 (동등 수용)

- 시험 결시
- 결시
- 시험 결시 처리
- 시험 병결
- 병결
- 병결/공결
- 시험 불참
- 시험 못 본 경우 / 시험 못봤을 때 (자연어 스타일)
- 결시 처리 / 결시 안내
- 영문 계열: `exam_absence`, `missed_exam`, `sick_leave_exam`

### 탈락 예

- `재수강`, `수강취소`, `학점포기` 같이 **다른 주제**로 새 인텐트를
  잡으려는 경우
- 완전 무관 (`장학금`, `도서관`, `교환학생`)
- 단일 단어 추상어 (`인텐트`, `새 인텐트`, `수정`)
- 무의미 문자열, 회피성 답변

## 트리거 문장 후보 (Add Triggering Sentence 작업 내용)

새 인텐트에 붙일 유저 발화 예:

1. **시험 못보면 어떻게 되는거야** ← 원본 오분류 사례
2. 시험 결시하면?
3. 시험 결시 처리
4. 시험 못 봤어요 / 시험 빠졌어요
5. 병결로 시험 못 봤는데
6. 시험 결시 사유 증빙

1~3번 정도가 최소 필수, 나머지는 운영 로그 보며 추가 Task로 보강.

## 향후 LLM 프롬프트에서 활용할 때

- `expectedTopic`: "시험 결시 / 병결 처리" (주제)
- `acceptableNames`: 위 "통과 가능한 네이밍 예" 리스트
- `acceptableTriggers`: 위 "트리거 문장 후보" 리스트
- 판정 기준은 제목 프롬프트와 동일한 결 — **대상 주제 일치 + 의미
  결합**이면 관대하게 통과, 완전 무관·무의미·추상어 단독은 탈락.
